## üí° Key Features & Interactivity

### 1. Product/Tech View Toggle
In the **Projects & Strategy** section, each project card features a dynamic toggle switch:
* **Product View:** Explains the **Problem, Strategy, and Business Impact**.
* **Tech View:** Details the **Architecture, Frameworks (Python, ML, APIs), and Implementation**.
* **Benefit:** Demonstrates the ability to communicate project value effectively to both technical and non-technical stakeholders.

### 2. Quantitative Impact Visualization
The **Experience & Impact** page converts achievement statements into visual metrics:
* A doughnut chart clearly illustrates the **~20% improvement in data load efficiency** via API integration.
* A separate chart highlights the **~30% reduction in UI bugs** achieved through code refactoring/debugging.

### 3. Competency Matrix
The **Skills** page uses a Radar Chart to map a balanced profile across six core domains:
* `Product Strategy`
* `AI/ML & Data`
* `Frontend/UX`
* `Backend/API`
* `Cloud/DevOps`
* `Agile/Process`

This visual summary quickly conveys the breadth of technical and strategic capabilities required for a Product Management role.
